工具介绍

实时监控和检测谁正在更改您的目录和网络共享。
目录监视器可用于监视目录和/或网络共享，并实时通知您文件更改、访问、删除、修改、新文件和不活动。 进行更改的用户和进程也可以被检测到。 目录监视器还提供文本日志、通过脚本/应用程序执行的自动化、电子邮件、写入数据库、声音通知、打印等

实时通知

在文件系统或网络共享上的文件修改、删除、重命名、新文件、文件访问和不活动状态的几毫秒内收到通知。

监视本地目录或网络共享，包括隐藏/私有共享。

用于检测子目录中的所有文件以及文件属性更改的选项。

每当检测到事件时就会发出气球通知。

每个目录包含和排除过滤器模式。

作为控制台应用程序 (PRO) 提供。

每秒能够捕获和处理超过 400 个更改事件。

准确检测用户进行更改（PRO）

近乎实时地找出哪些用户正在对目录或文件进行更改。

报告申请流程以及本地变更。

可以在本地或网络中的网络共享中检测用户。

自动配置计算机和目录以启用审核。

强制保留系统审核策略，即使恶意用户尝试禁用它

Ultimate reliability with snapshots (PRO)

Enable snapshots to ensure changes can be detected while the network is down and even during power outages.

Fast snapshots of directory and file statistics.

Ensure you never miss any changes as well as user and process detection.

Windows后台服务监控（PRO）

目录监视器可以安装为具有多个用户配置的 Windows 服务。

无需登录机器即可继续监控更改。

通过用户界面进行简单的配置和安装。

在后台运行时，内存和 CPU 占用量极低

事件发生时执行脚本/应用程序

当一个或多个文件系统更改事件发生时，可选择执行任何脚本或应用程序。

使用宏根据事件信息将上下文参数传递到脚本。

在后台静默执行脚本或应用程序 (PRO)。

如果脚本或应用程序不能运行多个实例，则为顺序执行选项。

根据事件数量、延迟或不活动进行批处理触发。

发送有关文件系统更改的电子邮件

针对每个事件或批量进行一定数量的更改后，轻松向多个收件人发送电子邮件。

灵活的配置选项可满足所有电子邮件服务器的需求。

使用宏提供基于事件的上下文信息。

基于事件数量、一段时间后没有事件、延迟或不活动的批处理发送规则。

当短时间内发生重大变化时进行限制以防止垃圾邮件

将事件保存到关系数据库

将文件系统事件存储在您选择的易于查询的关系数据库中。

支持 SQL Server、MySQL、PostgreSQL 和 Oracle。

自动创建所需的数据库模式。

播放声音以引起您的注意

发生变化时会收到音频通知。

每个文件系统事件都会播放一次声音。

声音可以连续循环播放，直到您采取行动并查看用户界面。

仅支持 WAV 文件。

Send file system event messages to a Syslog server

Send directory event messages to a Syslog server locally or on your network.

Default Syslog configuration options.

Support for both RFC3164 and RFC5424 Syslog header formats.

Define your own message format for each directory using macros.

Select any facility and priority level for your messages.

Growl 的高级通知

使用 Growl 在任何地方（甚至是您的移动设备）获取通知。

Growl 是一个强大的跨平台消息系统。

自定义通知警报窗口，比系统托盘气球侵入性更小且更可定制。

即使应用程序作为服务在后台运行，也能看到通知。

保留已通过的通知的历史记录。

本地通知或发送到远程目标（例如移动设备）的基本配置
